# 项目当前进度

最后更新：`2026-06-28`

## 当前判断

这个网站已经不处于“还没搭起来”的阶段。
现在更准确的状态是：主骨架已经成型，后续重点转向易用性、体验收口和长期维护。

公开站点、内容系统、结构化数据系统、评论系统、SEO 基础能力以及 `/studio` 内容管理链路都已经具备。
当前最主要的问题，不是缺页面，而是很多流程还“不够顺手”，尤其是 `/studio`。

## 已经具备的部分

### 站点结构
- 公开主页面已经具备：`首页`、`项目`、`博客`、`学习`、`关于`
- 次级公开页面已经具备：`说说`、`相册`、`友链`、`统计`、`成果`、`Now`
- 联系方式已经合并到关于页中

### 内容架构
- `博客`、`笔记`、`项目` 已具备基于 MDX 的内容系统
- `content/data` 下已具备结构化 JSON 数据系统
- 内容索引可通过 `pnpm content:generate` 自动生成
- 博客和笔记已经具备标签与关联结构基础

### 创作与后台
- `/studio` 已支持新建和编辑 MDX 内容
- `/studio` 已支持编辑结构化 JSON 数据
- `/studio` 已支持媒体上传、预览和部分批量操作
- 已部分适配 Obsidian 风格内容导入

### 发布与交付
- 已接入 `giscus` 评论
- 已具备基础 SEO、`sitemap`、`robots` 与 OG 能力
- 网站已经完成部署，并接入自定义域名
- 已建立第一版项目图谱与 AI 导航层：
  - `AGENTS.md`
  - `graphify-out/PROJECT_MAP.md`
  - `graphify-out/GRAPH_REPORT.md`

## 部分完成的部分

### `/studio` 易用性
- 功能已经较强，但日常创作的操作阻力仍然偏大
- 编辑链路偏长
- 内容浏览和筛选仍然不够流畅
- 状态反馈、批量操作、列表管理还需要继续打磨

### 视觉质量
- 整体风格已经可用，且大体统一
- 字体系统已经有基础，但还缺完整页面级 QA
- 移动端细节还没有完全收口

### 内容维护体验
- 内容与数据大体已经分离
- 但维护流程还没有达到“快速顺手”的程度
- 根目录 `README.md` 仍然保留着早期模板说明，没有反映当前项目真实情况

### 统计体验
- 已有访问统计能力
- 但查看和理解数据的运营体验还不够清晰

## 当前主要风险

1. `/studio` 功能强，但还不够适合高频发布使用
2. 在建立这些治理文档之前，项目推进记录偏碎片化
3. 仓库根目录 `README.md` 仍是模板初始内容
4. 仓库中还存在几处生成产物目录，后续需要评估是否清理：
   - `app/graphify-out/`
   - `components/graphify-out/`
   - `lib/graphify-out/`

## 近期最值得优先做的事

1. 让 `/studio` 在日常写作、编辑、媒体处理上更顺手
2. 优先提升桌面端与移动端的适配体验，而不是继续堆新功能
3. 统一项目文档与接手方式，减少未来重复分析成本
4. 把根目录 `README.md` 改成真实的项目说明

## 短期下一步

按照 `USABILITY_OPTIMIZATION_PLAN.md` 从第一阶段开始推进：
- 梳理 `/studio` 内容列表、筛选与编辑流
- 降低创作过程中的操作摩擦
- 提升保存、预览、内容组织的反馈清晰度
